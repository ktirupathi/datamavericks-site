3.1 · Feed Forward Networks

The Multilayer Perceptron

The simplest deep network — and the blueprint every other architecture is built on. One direction, no memory, layer by layer.

A feed forward network — also called a multilayer perceptron (MLP) — is the simplest deep neural network. Information flows in exactly one direction: from the input, through one or more hidden layers, to the output. No loops, no memory of previous inputs, no feedback. Every architecture you will meet later in this chapter — CNNs, RNNs, LSTMs, and ultimately Transformers — is a specialization or extension of this one idea.

The name is literal. Data is fed forward, one layer at a time. Each layer takes the previous layer's output, multiplies it by a matrix of weights, adds a bias, applies an activation function — the non-linearity — and hands the result to the next layer. Stack enough of these and the network can approximate essentially any continuous function. That guarantee has a name: the universal approximation theorem, and it is the theoretical reason MLPs matter at all.

01Building the intuition

A corporate approval chain

Analogy

Picture a loan application moving through a bank. A junior analyst sees the raw facts — income, credit score, loan amount — and forms an initial read. They pass a short summary up to a manager, who combines several analysts' reads into a higher-level judgment. The manager passes that to a director, who makes the final call.

Each level transforms what it received into something more abstract, and information only flows upward — the director never kicks the file back to the analyst mid-decision. That one-directional, layer-by-layer abstraction is exactly an MLP. The raw facts are the input layer, the analysts and managers are hidden layers building ever more useful representations, and the director is the output layer producing the answer.

02Mechanics

How it works

The atom of the network is a single neuron. It does two things, in order: a weighted sum of its inputs, then a transformation of that sum.

A single layer performs this for many neurons at once — which is just a matrix multiplication followed by an element-wise activation. A network chains layers together. The real work happens in the hidden layers: the input layer merely holds your raw features and the output layer produces the final answer, but the hidden layers in between learn intermediate representations — edges then shapes then objects in a vision network; syntax then meaning in a language one.

The math that matters

Formula · one neuron, then one layer
Single neuron:
   z = (w · x) + b           weighted sum
   a = f(z)                  activation (non-linearity)

Whole layer (vectorized):
   z⁽ˡ⁾ = W⁽ˡ⁾ a⁽ˡ⁻¹⁾ + b⁽ˡ⁾
   a⁽ˡ⁾ = f(z⁽ˡ⁾)

Forward pass = repeat layer-by-layer, input → output
W is the weight matrix for layer l, b its bias vector, f the activation. The output of one layer (a⁽ˡ⁻¹⁾) is the input to the next. That chaining is the entire feed forward computation.

Without the activation function f, this whole stack collapses. A chain of matrix multiplications is itself just one matrix multiplication — a thousand linear layers would equal a single linear layer, and the network could only ever draw straight decision boundaries. The non-linearity at each layer is what lets depth actually buy you expressive power. (This is why Chapter 2's activation functions come before architectures — they're the ingredient that makes depth meaningful.)

03Numbers on paper

Worked example — a 3→2→1 network

A tiny network that decides whether to approve a loan. Three inputs, one hidden layer of two neurons (ReLU), one output neuron (sigmoid, giving an approval probability).

Worked Example · forward pass
Inputs:   x = [income=0.7,  credit=0.6,  amount=0.4]

HIDDEN LAYER (2 neurons, ReLU)
 Neuron h1:  w=[0.5, 0.4, -0.2],  b=0.1
   z1 = 0.7(0.5) + 0.6(0.4) + 0.4(-0.2) + 0.1
      = 0.35 + 0.24 - 0.08 + 0.1 = 0.61
   a1 = ReLU(0.61) = 0.61

 Neuron h2:  w=[-0.3, 0.6, 0.5],  b=-0.2
   z2 = 0.7(-0.3) + 0.6(0.6) + 0.4(0.5) - 0.2
      = -0.21 + 0.36 + 0.20 - 0.2 = 0.15
   a2 = ReLU(0.15) = 0.15

OUTPUT LAYER (1 neuron, sigmoid)
   w=[0.9, 0.7],  b=-0.3
   z  = 0.61(0.9) + 0.15(0.7) - 0.3
      = 0.549 + 0.105 - 0.3 = 0.354
   y  = sigmoid(0.354) = 1 / (1 + e^-0.354)
      = 0.588   →   58.8% approval probability
Read the flow: three raw numbers became two hidden activations (a learned intermediate representation), which became one probability. Every value moved strictly left to right. Change any weight and the whole downstream result shifts — that is exactly what training adjusts. Try it live below.

05Why it matters

Why depth, and where MLPs stop

Stacking hidden layers lets the network compose simple features into complex ones. Layer one might learn to detect rough combinations of inputs; layer two combines those into higher-order patterns; and so on. This hierarchical feature learning is the whole reason "deep" learning is deep — and it is why a two-layer network often dramatically outperforms a one-layer network on the same data, despite the universal approximation theorem technically promising that even one wide hidden layer could fit the function. Depth is more parameter-efficient than width.

The limitation that drives the rest of this chapter

A plain MLP treats every input as an independent, fixed-size, unordered vector. It has no notion of space (a pixel's neighbors mean nothing to it) and no notion of sequence or memory (it can't tell "dog bites man" from "man bites dog"). Feed it a 200×200 image and you need 40,000 input weights per first-layer neuron — wasteful and blind to structure.

Those two gaps are exactly what the next architectures fix. CNNs (3.2) add spatial structure through shared filters. RNNs (3.3) add sequence and memory through recurrence. The MLP is the foundation; the rest of Chapter 3 is the story of giving it the senses it lacks.

The multilayer perceptron proved that stacking simple non-linear units could approximate anything. What it couldn't do — see space, remember order — is precisely why every architecture after it exists. On why the MLP is the starting point, not the destination

Bottom Line

The MLP is the foundation of deep learning: layers of weighted sums and non-linear activations, information flowing strictly forward. It can in principle approximate any function, and depth lets it do so efficiently by building a hierarchy of features. But it is blind to the structure of its inputs — it sees neither space nor sequence. Understand the MLP completely, because everything that follows in this chapter is an MLP given a new sense: CNNs see space, RNNs remember sequence, and Transformers learn what to pay attention to.
